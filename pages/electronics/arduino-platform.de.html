Die Arduino-Plattform - Eine Einführung

Vor rund zwei Monaten bin ich auf ein Projekt gestoßen - die Arduino-Plattform. Dabei handelt es sich um eine Mikrocontroller-Entwicklungsplatine, mit der man schnell und einfach elektronische Schaltungen entwickeln kann - selbst für Programmieranfänger.
Oder konkreter: Mit einem Arduino-Board kannst du ...

  • Sensordaten auslesen, z.B. Temperatur, Beschleunigung, Winkel, Kompass, Mikrofon, Ultraschall und viele mehr
  • dann diese Daten verarbeiten und mit ihnen weitere Geräte kontrollieren, z.B. einfache LEDs, Motoren, Lautsprecher, LCD-Displays etc.
  • Technologien benutzen wie Ethernet, WLAN, GSM, Bluetooth usw., um Informationen ins Internet zu übertragen
  • kreativ sein und die obigen Dinge beliebig kombinieren, um etwas komplett Neues zu schaffen
Aber zunächst einige Infos zum wichtigsten Teil jedes Arduino-Boards: Dem Mikrocontroller.

Was ist ein Mikrocontroller?

Ein Mikrocontroller (auch µC oder uC) ist im Wesentlichen eine "kleine" Ausgabe heutiger Computerprozessoren - er führt ein Programm aus, welches z.B. elektronische Geräte steuern oder Sensoren abfragen kann. Im Gegensatz zum PC-Prozessor enthält der Arduino-Mikrocontroller noch sogenannte "Peripherie": einen RAM-Baustein, um temporäre Daten im Speicher zu halten, einen Flashspeicher, der ein Programm enthält und noch einige andere. Zum Beispiel enthält der Prozessor des Arduino Leonardo, das ich zur Zeit benutze, einen USB-Chip, der direkt die Kommunikation per USB mit dem Computer übernimmt.

Der erste Sketch - Hallo Welt Blink

Um ein Programm auf einen µC zu laden, gibt es spezielle Geräte, sogenannte Programmer. Arduino-Boards gehen aber einen viel einfacheren Weg: Man schließt das Board mit USB an seinen Computer an, und dann können mit der Arduino-Software Programme (genannt Sketch) auf das Board geladen werden. Die Programmiersprache ist C++, welches sich für hardwarenahe Programmierung anbietet. Mit Wiring, einem Mikrocontroller-Framework, werden aber einige Details der Hardware versteckt. So sieht ein einfacher Sketch zum Beispiel so aus: 

int ledPin = 13;             

void setup() {
  pinMode(ledPin, OUTPUT);    // LED -> Ausgang
}

void loop() {
  digitalWrite(ledPin, HIGH); // LED anschalten
  delay(1000);                // 1 Sekunde warten
  digitalWrite(ledPin, LOW);  // LED ausschalten
  delay(1000);                // 1 Sekunde warten
}

Dieser Sketch lässt eine am Pin 13 (das sind die Anschlüsse auf dem Arduino-Board) angeschlossene LED im Sekundentakt blinken. In jedem Arduino-Sketch müssen die beiden Funktionen setup() und loop() vorhanden sein. Sobald das Board angeschaltet wird, führt der µC die Funktion setup() aus; sie enthält also meist typische Initialisierungsarbeiten. Hier wird der Pin, an dem die LED hängt, als Ausgang deklariert, damit die LED später an- und ausgeschaltet werden kann. Wie der Name schon andeutet, wird die Funktion loop() nach dem Ausführen von setup() fortlaufend wiederholt. Der Code in der Funktion schaltet die LED an, wartet 1 Sekunde, schaltet sie aus, wartet 1 Sekunde ... und so weiter, bis das Board von der Stromquelle getrennt wird.

Natürlich kann man mit Mikrocontrollern viel mehr machen als einige LEDs blinken zu lassen. Einige solcher Projekte von mir kannst du im Bereich Elektronik finden.

"Build apps that are a delight to use, faster than you ever thought possible"

Pretty bold headline, isn't it?

This is the current slogan of the Meteor Project. Meteor is a radical new way for quickly developing great web sites based on JavaScript. (Involving Node.js, Cordova/PhoneGap and "reactive" programming that just works.)

I highly recommend you to check it out (or rather Discover it), if you haven't yet. Especially if PHP gives you a headache (for good reasons) and Rails is too slow and bulky for your task.

Here's my personal opinion on Meteor:

It is great. I mean, seriously. Coding apps like this in a few hours, sometimes even minutes? Too good to be true.
Well, there are some drawbacks too. See it in my personal "Meteor pros and cons".

Start learning Meteor
Mind-blowing
  • + it's declarative. You tell Meteor what you want, and not how to do that (e.g. you tell Meteor there's a Posts collection and it wires up the whole CRUD business for you). This means you write less code in total
  • + it's reactive. This means that if some guy on the Canary Islands likes a post I created, this instantly shows up on my screen too without me writing code for that at all!
  • + there's latency compensation. Sounds rather complex, but it just means that the user doesn't need to wait on the server's response every time an API call is made (goodbye to many Loading ... messages)
  • + it has great community and documentation. Just look at the 1-page Docs. As for the community, people are incredibly helpful and there are such great apps built with Meteor (go check out some Lightning Talks)
Not So Great
  • - your client needs to enable JavaScript (unless you do some fancy stuff like Server Side Rendering)
  • - it's still tightly coupled to MongoDB. If you prefer other (e.g. SQL-esque) databases, you might run into issues
  • - meteor.com is fine for deployment at first, but not suitable for production (e.g. there's no oplog tailing which is vital to make your app fast) - objection: Galaxy is coming!
  • - no Windows support - objection: coming too!!1!
  • - I find Atmosphere (Meteor's package management server) to be a little confusing and crowded sometimes - how do I know which of the countless Bootstrap packages to choose? (there are official packages but I don't think they are distinctive enough)

Meteor is evolving at a rapid pace. This means some of the problems will be dealt with (mainly the Windows support).
But this can also be challenging because you have to keep up with its development - for that I recommend you to subscribe to crater.io, a Reddit-like newspaper for Meteor developers (speaking of which, there's a subreddit too).

So, that's what I think - now there's only one thing left to say: Start coding with Meteor, you'll love it! :)
